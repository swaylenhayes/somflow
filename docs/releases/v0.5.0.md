# v0.5.0 Release Notes

Fine-tuned YOLO detection model integrated into the pipeline. With `--yolo`, detection coverage on ScreenSpot-Pro jumps from 57.3% to 90.8% across 26 professional applications on macOS, Windows, and Linux.

## YOLO UI Detection (`--yolo`)

The `--yolo` flag enables a bundled YOLO11s model (18 MB) fine-tuned on GroundCUA (55K desktop screenshots, 3.56M human-verified annotations). The model detects buttons, menus, inputs, navigation elements, sidebars, and visual controls that Apple Vision misses.

```bash
# Vision-only (default, ~1s)
uitag screenshot.png -o out/

# Vision + YOLO (~5s, closes icon gap)
uitag screenshot.png --yolo -o out/
```

Install the YOLO dependency with `pip install uitag[yolo]` or `pip install ultralytics`.

### How it works

The model uses tiled inference (640x640 tiles, 20% overlap) to process high-resolution screenshots at the same scale it was trained on. Detections from all tiles are assembled with cross-tile NMS, then merged with Apple Vision's text and rectangle detections using IoU-based deduplication. Vision text labels take priority over YOLO class labels when the same element is detected by both.

### Detection Coverage

Measured on ScreenSpot-Pro (1,581 targets, 26 professional applications, 3 platforms):

| Mode | Text | Icon | Overall | Zero Detection |
|------|------|------|---------|----------------|
| Vision + YOLO | 92.7% | 87.6% | 90.8% | 5.8% |
| Vision-only | 66.4% | 42.5% | 57.3% | 32.9% |

The icon gap — 42.5% to 87.6% — is the headline improvement. Nearly a third of targets were previously invisible to Vision (32.9% zero detection rate), now reduced to 5.8%.

### Model Details

| Property | Value |
|----------|-------|
| Architecture | YOLO11s |
| Training data | GroundCUA tiled (224K tiles, 9 classes) |
| Training | 100 epochs, 2x H100 PCIe DDP, 19.75 hours |
| Model size | 18 MB |
| Validation mAP@0.5 | 0.792 |
| License | MIT |

The model is published on HuggingFace as `swaylenhayes/uitag-yolo11s-ui-detect-v1`.

## Pipeline Changes

- New module: `uitag/yolo.py` — tiled inference, cross-tile NMS, lazy model loading
- New source value: `"yolo"` in manifest elements (priority 2, same as `vision_rect`)
- New CLI flag: `--yolo`
- New Python API parameter: `run_pipeline(..., use_yolo=True)`
- Model weights bundled at `uitag/models/yolo-ui.pt`
- New optional dependency: `ultralytics` (install via `pip install uitag[yolo]`)

## Breaking Changes

None. The default pipeline (Vision-only) is unchanged. The `--yolo` flag is opt-in.

## Benchmark Summary

Six benchmark runs on ScreenSpot-Pro, three additional runs on GroundCUA and UI-Vision:

| Benchmark | Vision-only | YOLO-only | Vision+YOLO |
|-----------|-------------|-----------|-------------|
| ScreenSpot-Pro (all, 1,581) | 57.3% | 80.1% | 90.8% |
| ScreenSpot-Pro (macOS, 604) | 62.7% | 80.1% | 91.7% |
| GroundCUA (500) | 22.1% Recall | 94.0% | 89.9% |

## Requirements

- Python 3.10+
