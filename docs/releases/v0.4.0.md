# v0.4.0 Release Notes

## New Features

### Multi-Crop Ensemble OCR Rescan (`--rescan`)

Low-confidence text detections can now be re-scanned for improved OCR accuracy. This addresses a core reliability issue where Apple Vision's language correction post-processor corrupts correct character-level readings for code, regex, and variable text.

**How it works:** Elements below the confidence threshold (default 0.8) are cropped at 5 different padding values, re-OCR'd with language correction disabled, and the reading with the most special characters is selected. This compensates for Apple Vision's chaotic sensitivity to crop boundaries — shifting by just 5px can flip results from perfect to garbled.

**Research:** 8 phases of experiments validated this approach, including crop boundary sensitivity analysis, image pre-processing gradients, context-swap tests, and light/dark mode comparison. See [`docs/research/ocr-rescan-experiments.md`](../research/ocr-rescan-experiments.md) for full findings.

**Low-confidence callout (default):**

When running `uitag`, the CLI now automatically flags low-confidence detections:

```
Done: 151 detections in 2.4s

! 3 low-confidence detection(s):
  [7]  ";([\w_l+);"  conf: 0.30
  [27] ": (Ow_]+);"  conf: 0.50
  [4]  "xmi-wrap-word"  conf: 0.85

Tip: run with --rescan to re-check at higher resolution
```

**Rescan flag:**

```bash
# Rescan all low-confidence text
uitag screenshot.png --rescan

# Rescan specific elements by SOM ID
uitag screenshot.png --rescan 7,27
```

**Light mode note:** OCR accuracy is measurably better on light mode screenshots, especially for thin characters like `\` in regex patterns. If accuracy is critical, consider capturing in light mode. See [Tips](#tips) in the README.

### Patch JSON Input (`uitag patch`)

Re-annotate images from modified manifest data without re-running the detection pipeline. This enables agent-driven correction workflows — an agent reads the manifest, identifies OCR errors, and provides a patch file with corrections.

```bash
# Apply corrections from a patch file
uitag patch screenshot.png --manifest manifest.json --patch corrections.json -o out/

# Patch file format
{
  "patches": [
    {"som_id": 7, "label": ";([\\w_]+);"},
    {"som_id": 4, "label": "xml-wrap-word"},
    {"som_id": 12, "hide": true}
  ]
}
```

Supported patch operations:
- `label` — Replace the detected text
- `confidence` — Update the confidence score
- `hide` — Remove the element from the output

### Render from Manifest (`uitag render`)

Render annotations on an image from an existing manifest without re-running detection. Useful for re-rendering after manual manifest edits or for applying a manifest to a different image.

```bash
uitag render screenshot.png --manifest manifest.json -o out/
```

## Pipeline Changes

- New optional stage 4b (rescan) between merge and annotate
- Rescan uses multi-crop ensemble: 5 padding values, special-character heuristic selection
- Apple Vision `--no-lang-correction` flag for rescan pass (accurate mode, language correction disabled)
- Rescan timing and stats included in manifest `timing_ms` when `--rescan` is used
- Three new `run_pipeline` parameters: `rescan`, `rescan_threshold`, `rescan_ids`

## New Modules

| Module | Purpose |
|--------|---------|
| `uitag/rescan.py` | Multi-crop ensemble OCR rescan logic |
| `uitag/patch.py` | Patch application and manifest loading |
| `uitag/patch_cli.py` | `uitag patch` and `uitag render` subcommands |

## Known Limitations

- Rescan uses Apple Vision only (no Florence-2 re-detection for non-text elements)
- Multi-crop ensemble adds latency: ~250-500ms per low-confidence element (5 Vision calls on ANE)
- Dark mode screenshots produce lower OCR accuracy for thin special characters (`\`, `;`, `[`) — light mode is recommended for best results
- Patch does not re-run deduplication — applying patches that change bounding boxes may create overlaps
